# ADR-0006. Внешний доступ: поддомен DuckDNS + Let's Encrypt

Дата: 2026-07-11. Статус: принято (решение владельца, Фаза 4).

## Контекст

Умному дому Яндекса нужен публичный HTTPS с валидным сертификатом
(план §6); своего домена у проекта не было. Варианты:

- **A** — IP-сертификат Let's Encrypt: короткоживущий (~6 дней, профиль
  `shortlived`), плюс неопределённость — примет ли консоль Диалогов
  Endpoint URL с голым IP;
- **B** — бесплатный поддомен DuckDNS + обычный certbot;
- **C** — платный домен (самый надёжный, но стоит денег).

## Решение

Путь **B**: поддомен `*.duckdns.org` указывает на IP VPS; обычные
90-дневные сертификаты Let's Encrypt с автопродлением; TLS терминирует
nginx на VPS перед frp-туннелем (deploy/nginx). Процедура —
`docs/yandex-setup.md`, шаг 0.

## Последствия

- Бесплатно; без короткоживущих сертификатов и без ставки на поддержку
  IP-URL консолью Диалогов.
- Зависимость от сервиса DuckDNS (без SLA) — приемлемо для приватного
  навыка; IP VPS статичен, так что обновлятор DNS не нужен.
- Миграция на путь C при желании тривиальна: A-запись, перевыпуск
  сертификата, замена `server_name` и Endpoint URL/OAuth-адресов в консоли
  Диалогов; код и конфигурация сервиса не меняются.
