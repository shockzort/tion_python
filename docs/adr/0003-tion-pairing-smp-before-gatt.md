# ADR-0003. Сопряжение Tion 4S: SMP до GATT, самодостаточный агент

Дата: 2026-07-11. Статус: принято (Фаза 1, полевые факты — спека §1.7/§5).

## Контекст

Прошивка 4S принимает LE-соединения только от сопряжённых центральных;
незнакомый `CONNECT_IND` игнорируется молча (HCI 0x3E). Новый центральный
подключается только в режиме сопряжения (кнопка ~5 с; байт в рекламе
`00`→`01`). В режиме сопряжения бризер рвёт соединение, если центральный
начинает GATT-discovery раньше SMP. Без `Pairable: yes` и агента BlueZ
заваливает SMP (`AuthenticationFailed`) — на headless RPi «подпорки»
вроде bluetoothctl не будет.

## Решение

`BleakTransport.pair()` выполняет сопряжение атомарно и самодостаточно:

1. Найти устройство сканом (объект BlueZ).
2. Поднять JustWorks-агент (`org.bluez.Agent1`, NoInputNoOutput) и включить
   `Pairable` на адаптере (запомнив прежнее значение).
3. `Device1.Pair()` на объекте устройства — BlueZ сам делает connect → SMP
   в правильном порядке; `AlreadyExists` считается успехом (идемпотентность).
4. Вернуть адаптер как было, подключить транспорт (соединение уже поднято).

Агент — отдельный модуль `ble/_bluez_agent.py` без
`from __future__ import annotations` (dbus-fast читает сырые аннотации).

## Последствия

- `breezy pair` работает на чистой системе без bluetoothctl (RPi-ready).
- Сопряжение — разовая процедура на пару «хост × бризер»; бонд хранит BlueZ.
- `Pair()` вне режима сопряжения даёт ложный `Page Timeout` — диагностика
  описана в `docs/runbook.md`.
