# ADR-0004. Записи в устройство подтверждаются перечитыванием

Дата: 2026-07-11. Статус: принято (Фаза 1, полевые факты — спека §1.6).

## Контекст

SET_PARAMS задаёт полный набор параметров разом (не дельту). Устройство
присылает кадр состояния после применения (~150 мс), но: кадр может
потеряться при сборке (флаки BLE), прошивка вправе скорректировать
запрошенные поля (наблюдение: включение рециркуляции меняло fan), а внешние
контроллеры (MagicAir) перезаписывают состояние независимо от нас.

## Решение

- Дельта мержится поверх актуального состояния вызывающим
  (read-modify-write); драйвер шлёт только полный набор.
- `S4Driver.set_state()` после кадра подтверждения (или тишины 3 с) всегда
  выполняет контрольный `REQUEST_PARAMS` и возвращает фактическое состояние.
- Сверка «применилось ли» — ответственность вызывающего (command bus
  Фазы 2): сравнить фактическое с желаемым, решить о ретрае/эскалации.

## Последствия

- +1 RTT (~150 мс) на каждую запись — приемлемо при цели < 2 с.
- Драйвер никогда не возвращает «свою веру» — только истину устройства.
