# ADR-0002. Слоистая BLE-архитектура и FakeS4 для разработки

Дата: 2026-07-11. Статус: принято (Фаза 1).

## Контекст

BLE-обмен с Tion — самая хрупкая часть системы: радио, чужие централы,
однобайтовые протокольные ловушки (инверсия бита нагрева). Нужны
тестируемость без железа и изоляция протокола от I/O.

## Решение

Слои в `server/src/easy_breezy/ble/`:

1. `protocol/` — чистые кодеки без I/O и случайности (nonce инжектируется);
   обязаны байт-в-байт проходить golden-векторы (`server/tests/golden/`).
2. `transport.py` — `BleTransport` (Protocol) + `BleakTransport` (BlueZ);
   экземпляр одноразовый, паттерн «find_device → connect по объекту».
3. `driver.py` — `S4Driver`: кадры запрос/ответ, слушатель нотификаций,
   per-device сериализация операций.
4. `supervisor.py` — переподключение (backoff 1→60 с + джиттер), опрос,
   деградация; время инжектируется (time-travel тесты).
5. `fake.py` — `FakeS4Device`/`FakeTransport` с инъекцией сбоев: юнит-тесты
   и dev-режим `EB_FAKE_DEVICES=3` (сервис и UI разрабатываются без железа).

## Последствия

- BLE-железо в тестах запрещено; вся логика проверяется на fake.
- Новые модели (Lite, S3) добавляются кодеком + параметрами драйвера,
  транспорт/супервизор не меняются.
