## Осталось (операционные хвосты, шаги владельца)

### 1. Вернуть модуль CO2+ в приложение MagicAir

Фаза 8 закрыта вариантом **A**: источник CO2+ остаётся облачным
MagicAir (реализовано в Фазе 6, работает). При разведке модуль был
штатно удалён из приложения — сейчас облачных данных CO2+ нет. Вернуть:
включить базовую станцию → приложение MagicAir → «Доступные устройства»
→ «Добавить» → «Модуль CO2+» → кнопка инициализации на обороте →
«Готово». После этого станция и модуль снова появятся в Easy Breezy
(автосинк реестра). Почему прямой BLE отпал — `docs/protocol/tion-co2plus-ble.md`.

### 2. APK и «Опасный сайт» — дождаться снятия флага Google

Свежий `mobile/easy-breezy-1.0.0.apk` установлен. Бровку «Опасный сайт»
рисует Chrome из-за вердикта Safe Browsing на `easy-breezy.duckdns.org`
(TWA-верификация в порядке, `assetlinks.json` проверен — виновата
репутация бесплатного dyndns, не эмбеддинг). Сделано: заявка о ложном
срабатывании подана, Search Console подтверждён
(`ui/public/google79357c1087efb183.html`). Осталось: дождаться проверки
Google (1–3 дня; статус — в «Проблемах безопасности» Search Console);
после снятия бровка исчезает сама. Надёжно навсегда — свой домен вместо
duckdns (код домен не хардкодит; переделки малые — детали ниже в бэклоге).

### 3. Недельный soak на NUC

RSS контейнера < 300 МБ (NFR) за неделю. BLE-coexistence сейчас не
риск — Wi-Fi на NUC отключён, хост по LAN; если вернёте Wi-Fi и пойдут
0x3E — диагностика в runbook. Бризеры на NUC уже сопряжены.
