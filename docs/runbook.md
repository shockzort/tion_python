# Runbook Easy Breezy

Наполняется по фазам. Сейчас: hardware-смоук Фазы 1.

## Hardware-смоук Фазы 1 (реальный Tion S4, машина с BLE-адаптером)

Установка: `cd server && uv sync`, команды — через `uv run breezy …`
(или `breezy` после `uv tool install .`). Флаг `-v` включает DEBUG-логи BLE.

| # | Шаг | Ожидание | Уточняем в спеке |
|---|---|---|---|
| 1 | `breezy scan` (бризер включён) | бризер виден: имя, MAC, RSSI | точный формат имени → §1.1 |
| 2 | `bluetoothctl` → `connect <MAC>` → `menu gatt` → `list-attributes` | сервис `98f00001-…`, характеристики `…0002`/`…0003` | подтверждение UUID → §1.1 |
| 3 | бризер в режим сопряжения (по инструкции Tion) → `breezy pair <MAC>` | «Сопряжение выполнено» + вывод состояния | |
| 4 | `breezy state <MAC>`; рядом открыть приложение Tion | все поля совпадают с приложением | |
| 5 | `breezy set <MAC> --fan N` для N=1…6 | смена оборотов на слух, подтверждение в выводе | |
| 6 | `breezy set <MAC> --heater`, затем `--no-heater` | **сверить с приложением Tion — бит инвертирован** (спека §1.5/1.6) | |
| 7 | `breezy set <MAC> --temp 22` | цель 22 °C в приложении | |
| 8 | `breezy set <MAC> --mode recirculation`, затем `--mode outside` | заслонка слышна/видна в приложении | |
| 9 | `breezy set <MAC> --sound --light`, затем `--no-sound --no-light` | реакция устройства | |
| 10 | `breezy monitor <MAC>` → выдернуть питание бризера → подать | `[нет связи]` → `[на связи]` < 90 с, без перезапуска команды | |
| 11 | `breezy -v set <MAC> --fan 3` несколько раз | p95 «запись→кадр состояния» < 2 с (по таймстампам ble_tx/ble_rx) | |
| 12 | в `-v` логах после SET | пришёл ли state-кадр без запроса (гипотеза §1.6); нет ли `crc_mismatch` (→ CRC подтверждён, §4 п.3–4) | |

Результаты шагов 1, 2, 12 внести в `docs/protocol/tion-s4-ble.md` (статусы [u]/[?] → [✓]).
