# Модуль CO₂+ MagicAir (SP320) — разведка BLE (Фаза 8)

Статус: **прямой BLE недоступен** (R&D 2026-07-12 на боевом NUC).

## Вывод

Модуль CO₂+ (модель SP320) **не имеет BLE-интерфейса для прямого
подключения** — в отличие от бризеров S4 (у тех GATT-сервис
`98f00001-…`). Исходная постановка Фазы 8 («CO₂+ добавляется как бризер,
кнопка спаривания, мастер в UI, чтение поверх `BleTransport`») на этом
железе **неисполнима**. Решение по замене — за владельцем (`docs/status.md`).

## Чем подтверждено

**Документация (официальный РЭ SP320,
`pritokvozduha.ru/download/RE-Tion_CO2_16101418.10.2016.pdf`):**
- Модуль — «блок связи с Базовой станцией Тион MagicAir»; «передаёт данные
  на Базовую станцию»; «не имеет выхода в Интернет».
- Добавление идёт **через базовую станцию**: приложение MagicAir →
  «Доступные устройства» → «Добавить» → «Модуль CO₂+ SP320» →
  кратковременно нажать кнопку инициализации на обороте → приложение:
  «идёт подключение Модуля CO₂+ **к Базовой станции**». Телефон только
  инициирует; endpoint для стороннего центрального не появляется.
- На обороте две кнопки: инициализация (первичное соединение со станцией)
  и сброс к заводским. Индикация «случайные светодиоды» = модуль запущен.
- Питание — USB 5 V / 1 A от сети (не батарея); своего «вкл/выкл» нет.
- Связь со станцией — «по радиоканалу», тип в РЭ не назван. Открытые
  источники: бризеры Tion — по BLE, а вспомогательные устройства
  (Clever MAC и т. п.) — по RF 868 МГц; CO₂+ в этот же RF-класс.

**Эмпирика (адаптер NUC, Intel; сканер BlueZ активный, оба PHY —
LE 1M + LE Coded, `PHYs 0x05`, дедуп выключен):**
- Серия сценариев: короткое/долгое нажатие основной кнопки, потайная
  «технологическая» кнопка (сброс), power-cycle питания, штатное удаление
  модуля из приложения + режим инициализации по кнопке. Непрерывный
  захват рекламы + `btmon` HCI на хосте.
- Модуль **ни разу не появился в BLE-эфире**: ни имени (`tion|co2|magic`),
  ни company ID `0xFFFF` (как у S4), ни сервисов `98f0…`.
- MAC модуля из облака MagicAir (`7E:59:7C:37:39:57`) и MAC базовой
  станции (`71:4B:72:30:2F:50`) не встретились ни в одном HCI-дампе —
  ни в прямом, ни в обратном порядке байт.

## Урок

Контрольный эксперимент против гипотезы (ср. [[ble-adapter-0x3e-issue]]):
предположение «CO₂+ = бризер по BLE» опровергнуто и документацией, и
эфиром до написания драйвера. Golden-векторов нет — снимать нечего.

## Восстановление после разведки

Модуль был **штатно удалён из приложения** (отвязан от станции). Чтобы
вернуть облачный источник CO₂+: включить базовую станцию → в приложении
MagicAir «Доступные устройства» → «Добавить» → «Модуль CO₂+» → кнопка
инициализации на обороте → «Готово».
