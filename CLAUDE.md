# Конвенции разработки — Tion Python

## Проект

Python-сервис управления бризерами Tion через BLE с интеграцией Яндекс Алисы. Сервер работает на Raspberry Pi / Linux. Взаимодействие с устройствами — через библиотеку TionAPI поверх Bleak (asyncio BLE).

## Технологический стек

- Python 3.10+, asyncio
- Bleak (BLE), TionAPI
- Flask или FastAPI (HTTP API / веб-интерфейс)
- SQLite (хранение), Redis (кеширование, опционально)
- MQTT (сторонние датчики)
- pytest, pytest-asyncio, pytest-mock

## Стиль кода

**Форматирование:**

- `black` — автоформатирование, длина строки 88.
- `isort` — сортировка импортов, профиль `black`.

**Линтинг:**

- `ruff` — основной линтер (заменяет flake8 + flake8-bugbear).
- `mypy` — статическая типизация, строгий режим (`strict = true`).

**Типизация:**

- Аннотации типов обязательны для всех публичных функций и методов.
- Использовать `from __future__ import annotations` для отложенных аннотаций.
- Предпочитать `X | None` вместо `Optional[X]` (Python 3.10+).

**Docstrings:**

- Google-style для всех публичных методов и классов.
- Приватные методы — по необходимости.

## Именование

- `snake_case` — переменные, функции, модули.
- `PascalCase` — классы.
- `UPPER_SNAKE_CASE` — константы.
- Исключения — суффикс `Error` (например, `BleConnectionError`).
- Булевы переменные — префикс `is_` / `has_` / `can_`.
- Асинхронные функции не требуют префикса `async_` — имя должно отражать действие.

## Работа с asyncio и BLE (Bleak)

- Все BLE-операции — асинхронные, использовать `async with BleakClient(...)`.
- Никогда не блокировать event loop: запрещены `time.sleep`, синхронные I/O в корутинах.
- Для CPU-bound операций использовать `asyncio.to_thread` или `loop.run_in_executor`.
- Переподключение при потере связи — экспоненциальная задержка (начиная с 1 сек, максимум 60 сек).
- Все BLE-исключения (`BleakError` и подклассы) перехватывать явно, не ловить `Exception` вслепую.
- Таймауты на BLE-операции выставлять всегда.

```python
async def connect_device(device: BLEDevice, timeout: float = 10.0) -> None:
    async with BleakClient(device, timeout=timeout) as client:
        await client.write_gatt_char(CHAR_UUID, data)
```

## Тестирование

- Фреймворк: `pytest` + `pytest-asyncio` + `pytest-mock`.
- Покрытие: целевое > 80% (`pytest-cov`).
- Все BLE-вызовы и внешние API мокировать через `mocker.patch`.
- Асинхронные тесты декорировать `@pytest.mark.asyncio`.
- Тесты не должны требовать реального BLE-оборудования.

```python
@pytest.mark.asyncio
async def test_device_connect(mocker):
    mock_client = mocker.patch("bleak.BleakClient.connect")
    await connect_device(fake_ble_device)
    mock_client.assert_called_once()
```

## Git-конвенции

**Формат коммита:**

```text
[тип](область): описание
```

**Типы:**

- `feat` — новая функциональность.
- `fix` — исправление ошибки.
- `improvement` — улучшение без нового функционала.
- `refactor` — рефакторинг без изменения поведения.
- `test` — добавление или правка тестов.
- `docs` — изменения в документации.

**Ветки:**

- `master` — стабильная версия.
- `feature/<описание>` — новый функционал (например, `feature/ble-retry-logic`).
- `fix/<описание>` — исправление (например, `fix/web-auth`).

## Безопасность

- Секреты (токены, пароли, ключи) — только в `.env`, файл в `.gitignore`.
- Загрузка через `python-dotenv`, не хардкодить значения в коде.
- Зависимости: фиксировать версии в `requirements.txt`, регулярно проверять через `pip-audit`.
- Пароли хранить только в виде хешей (bcrypt/argon2).
- HTTPS обязателен для внешнего доступа (Nginx + Let's Encrypt).

## Логирование

- Использовать стандартный `logging` или `structlog`.
- JSON-формат логов (для возможной интеграции с ELK/Graylog).
- Уровни:
  - `DEBUG` — детали BLE-операций.
  - `INFO` — штатные события (подключение, выполнение команды).
  - `WARNING` — нештатные, но не критичные ситуации.
  - `ERROR` — сбои, требующие внимания.
- Ротация логов по размеру или времени.
- BLE-события логировать в отдельный файл.

## Запрещённые паттерны

- `except Exception:` без повторного raise или явного логирования — запрещено.
- `time.sleep()` в асинхронном коде — запрещено (использовать `asyncio.sleep`).
- Хардкод секретов, MAC-адресов, IP в коде — запрещено.
- Глобальный mutable state — запрещено (передавать зависимости явно).
- `print()` вместо `logging` — запрещено в production-коде.
- Синхронные HTTP-запросы (`requests`) в asyncio-контексте — запрещено (использовать `httpx` или `aiohttp`).
- Коммиты напрямую в `master` без проверки — запрещено.
