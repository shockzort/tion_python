# Business Requirements Document (BRD) — интеграция бризеров Tion с Яндекс Алисой

## 1. Цели проекта

- Голосовое управление бризерами Tion через Яндекс Алису.
- Автоматизация вентиляции на основе данных с датчиков CO₂ и других датчиков.
- Снижение зависимости от мобильного приложения Tion.

## 2. Ожидаемые результаты

**Функциональные:**

- Голосовые команды через Яндекс Алису:
  - Включение/выключение бризеров.
  - Переключение режимов работы ("тихий", "турбо" и др.).
  - Установка скорости вентиляции для конкретного бризера.
  - Управление подогревом: включение/выключение, регулировка температуры.
  - Запрос текущего состояния (температура, CO₂, режим).
- Интеграция с 3 бризерами и 2 датчиками CO₂ через TionAPI.
- Интеграция со сторонними датчиками (температура, влажность).
- Групповые команды (управление несколькими бризерами одновременно).
- Настройка сценариев ("ночной режим", "проветривание" и др.).
- Веб-интерфейс для управления (оффлайн).

**Нефункциональные:**

- Задержка выполнения команд < 3 секунд.
- Совместимость с Linux-сервером (Raspberry Pi или аналог) с Bluetooth 4.0+.
- Безопасное хранение данных аутентификации.
- Персистентное хранение известных устройств.

## 3. Заинтересованные стороны

| Роль               | Описание                                          |
| ------------------ | ------------------------------------------------- |
| **Владелец дома**  | Инициатор проекта, конечный пользователь.         |
| **Разработчик**    | Голосовая LLM модель, реализующая функционал.     |
| **Яндекс (Алиса)** | Голосовой интерфейс и средства интеграции.        |

## 4. Контекст и область применения

**Текущая ситуация:** управление бризерами возможно только через приложение Tion, которое работает через нестабильную базовую станцию. Голосовой доступ отсутствует.

**Область применения:** частные дома и квартиры с бризерами Tion. Сценарии: регулировка при готовке, автопроветривание по CO₂, голосовое управление при ограниченной мобильности.

**Ограничения:**

- Радиус Bluetooth до 10 метров без препятствий.
- Зависимость от Яндекс Алисы.
- Требуется постоянное питание сервера.

## 5. Технический контекст

**Инфраструктура:**

- Сервер (Raspberry Pi 4 / Orange Pi 5 / AMD) на Linux, Python 3.10+.
- BLE-адаптер.
- Локальная сеть с доступом в интернет.

**Компоненты:**

- Модуль взаимодействия с TionAPI.
- Навык Яндекс Диалогов (Skill для Алисы).
- Веб-интерфейс управления (оффлайн / Android).
- БД для хранения настроек (SQLite).
- Логирование ошибок и статуса устройств.

## 6. Риски

- Нестабильность Bluetooth-соединения.
- Изменения API Яндекс без обратной совместимости.
- Отсутствие официальной документации TionAPI.
- Лимиты запросов к Алисе.
