# Functional Requirements Document (FRD) — интеграция бризеров Tion с Яндекс Алисой

## 1. Функциональные требования

### 1.1. Управление устройствами

**Голосовые команды через Яндекс Алису:**

- Включение/выключение бризеров по имени (например, "включи бризер в гостиной").
- Установка скорости вентиляции (1–6 уровней).
- Переключение режимов: "тихий", "турбо", "ночной", "проветривание".
- Управление подогревом: включение/выключение, установка температуры (10–30°C).
- Запрос статуса: температура, CO₂, влажность, режим работы.

**Групповое управление:**

- Команды для всех устройств ("выключи все бризеры").
- Создание групп устройств через веб-интерфейс.

**Интеграция с датчиками:**

- Автоматическая корректировка работы бризеров:
  - CO₂ > 1000 ppm → активация "турбо"-режима.
  - Температура > 25°C → отключение подогрева.
- Поддержка сторонних датчиков (через MQTT/HTTP API).

### 1.2. Сценарии автоматизации

- Триггеры по времени (например, "ночной режим с 23:00 до 7:00").
- Триггеры по показаниям датчиков (CO₂, температура, влажность).
- Хранение сценариев в БД с возможностью редактирования.

### 1.3. Веб-интерфейс

- Поиск, добавление и удаление устройств.
- Просмотр статуса устройств в реальном времени.
- Ручное управление (режимы, скорость, температура).
- Настройка сценариев автоматизации.
- История изменений и статистика.
- Визуализация данных с датчиков за период.
- Локальный доступ (оффлайн) и удалённый доступ через HTTPS с авторизацией.

## 2. Нефункциональные требования

**Производительность:**

- Задержка выполнения команд: < 3 сек.
- Поддержка до 10 одновременных пользователей.

**Расширяемость:**

- Возможность интеграции других устройств умного дома (датчики температуры, влажности, протечки, газа и др.).

**Совместимость:**

- ОС: Linux (Debian/Ubuntu/Raspberry Pi OS).
- Python 3.10+.
- Браузеры: Chrome, Firefox, Safari.

**Безопасность:**

- Шифрование данных аутентификации (AES-256).
- Регулярное обновление сертификатов TLS.

**Надёжность:**

- Автоматическое переподключение при потере связи с устройством.
- Ежедневное резервное копирование БД.

## 3. Технологический стек

- Python: TionAPI, Flask/FastAPI.
- BLE: библиотека Bleak.
- MQTT: интеграция сторонних датчиков.
- БД: SQLite (хранение), Redis (кеширование).

## 4. Bluetooth-сопряжение

- Сканирование BLE-устройств с фильтрацией по имени ("Tion_Breezer_*").
- Сохранение MAC-адресов в БД после первого подключения.
- Авторизация через статический PIN-код (из документации Tion).

## 5. Пользовательский интерфейс

**Веб-интерфейс:**

- Главная страница: список устройств с кнопками управления и статусом.
- Сценарии: форма создания/редактирования.
- Настройки: добавление устройств, управление пользователями.
- Статистика: графики CO₂, температуры за выбранный период.
- Адаптивный дизайн для мобильных устройств.

## 6. Развёртывание

- Docker-контейнеры для изоляции компонентов.
- systemd-сервис для автоматического запуска.
- Nginx как reverse proxy с HTTPS (Let's Encrypt).
- CI/CD через Git с автоматическим тестированием и деплоем.

## 7. Обработка ошибок и логирование

- Хранение логов в файлах с ротацией по размеру/времени.
- Фиксация ошибок Bluetooth, API, пользовательских действий.
- Уведомления при критических сбоях (email, пуш через Android-приложение).
